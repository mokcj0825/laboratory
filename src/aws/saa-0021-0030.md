- S3 Standard: highly available, well-round services
- S3 Standard Infrequent Access: Cheaper, but weak in unpredictable access
- S3 One Zone Infrequent Access: Cheapest, but no AZ redundancy
- S3 Intelligent-Tiering: Automatically moves objects between frequent and infrequent access tiers
- S3 Glacier Deep Archive: cheapest for long-term, rarely accessed data
- S3 Intelligent-Tiering: Good, but need monitoring fee, if access patterns predictable, no worth or no need on it.
- S3 Standard Infrequent Access: More expensive than Glacier Deep Archive for long-term storage
- S3 One Zone Infrequent Access: Not multi-AZ, fails durability/availability for backups.
- Cost Explorer provides Graphs for last 2 months of costs, with granular filters.
- AWS Budgets good for alert, not deep cost analysis.
- Billing Dashboard too simplified.
- Cost and Usage Report + QuickSight need heavy setup.
- EC2 + Tomcat requires high ops burden
- DynamoDB + DynamoDB Accelerator not appropriate if we use RDBMS, we need change schema.
- SNS just one-way messaging.
- AWS Config continuously monitors and record AWS resource configurations
- Trusted Advisor gives best practice checks
- Amazon Inspector focus on EC2 instance vulnerabilities
- Server access logging + EventBridge tracks access events.
- CloudWatch support built-in direct dashboard with shareable URL
- AWS SSO will trust Microsoft AD for authentication, so no need two-way trust.
- RDS instance not support autoscaling
